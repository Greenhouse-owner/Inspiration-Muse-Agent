# oiioii Muse — 技术开发文档

## 1. 产品概述

### 1.1 产品定位

oiioii Muse 是一款**灵感快速生成器**。MVP 阶段以展示网页形态发布，用户可以在网页上直接体验：点击右下角几何桌宠，弹出 IM 风格对话框，通过选词（而非打字）的方式快速组合灵感元素，一键生成 200-300 字的故事情节梗概。

后续扩展为浏览器插件，让用户在任意网页上随时召唤 Muse。

核心理念：**选比写容易，组合产生惊喜。**

### 1.2 MVP 目标

> 验证：用户是否愿意通过"选词组合"的方式获得创意灵感，而不是从零开始构思。

### 1.3 核心使用场景

```text
用户访问 oiioii Muse 展示网页
→ 浏览产品功能介绍和技术架构
→ 点击右下角几何桌宠
→ 弹出 IM 对话框
→ 词云上方独立方向条显示当前方向（默认「角色设计」，共 4 个方向）
→ 词云区出现 15-25 个相关词：90% 当前方向词 + 10% 跨界惊喜词
→ 用户可点击 ◀ ▶ 或方向条本身切换方向（角色 / 场景 / 世界观 / 故事梗概）
→ 切换方向时词云立即重排，已选词保留在锁定栏并反向影响新词云内容
→ 用户在词云中点选词汇，已选词进入锁定栏（跨方向常驻）
→ 点击 🔄 换一批可在当前方向内刷新词云（不动方向条）
→ 也可以自己输入词汇加入已选列表
→ 选够 3-20 个词后，点击生成按钮
→ 对话框里以聊天消息形式出现一段故事梗概（200-300 字）
→ 用户可以在对话框里继续修改、追加要求
```

## 2. MVP 功能范围

### 2.1 P0 必做功能

```text
1. 展示网页上线（产品介绍 + 可交互体验）
2. 网页顶部：产品名称 + slogan
3. 网页中间上半部分：产品功能介绍卡片
4. 网页中间下半部分：技术架构介绍卡片
5. 网页右下角：可交互几何桌宠
6. 点击桌宠弹出 IM 对话框
7. 对话框内独立方向条（4 个方向：角色设计 / 场景设计 / 世界观设计 / 故事梗概设计）
8. 方向条支持左右箭头切换、点击循环切换；切换即触发词云重排
9. 词云区展示 15-25 个大小不等的词汇标签
10. 词云内容 = 90% 当前方向词 + 10% 跨界惊喜词（带角标，点击即跳转到对应方向）
11. 标签来源：本地词库 + AI 动态生成（按方向分别走对应词库与 prompt）
12. 用户点击标签选中（高亮）
13. 已选标签进入顶部锁定栏，跨方向常驻，点击 ✕ 可取消
14. 已选词反向影响新方向的词云内容（前端传 selectedTags 给后端，后端结合 direction 生成）
15. 🔄 换一批按钮：在当前方向内刷新词云，不动方向条与锁定栏
16. 用户可手动输入自定义词汇加入已选列表
17. 选词数量限制：最少 3 个，最多 20 个
18. 生成按钮：基于已选词汇 + 经过的方向上下文生成故事梗概
19. 故事梗概以 IM 消息气泡形式出现在对话框中
20. 用户可在对话框中继续输入修改需求
```

## 3. 技术选型

### 3.1 前端（展示网页）

```text
框架：React 18
开发语言：TypeScript
构建工具：Vite
样式：Tailwind CSS
配色：暗色主题 + 品红色（#FF2D78）点缀
动效：CSS Animation（桌宠呼吸发光、状态切换）
本地存储：localStorage
部署：Vercel / Netlify
```

### 3.2 后端服务

```text
后端框架：FastAPI
开发语言：Python 3.11+
数据校验：Pydantic
HTTP 请求：httpx
服务运行：uvicorn
环境变量：pydantic-settings
部署：Railway / Render / Fly.io
```

### 3.3 技术选择理由

| 模块 | 选择 | 原因 |
|---|---|---|
| React SPA | 前端框架 | 组件化清晰，适合对话框和标签 UI，部署简单 |
| TypeScript | 类型安全 | 降低标签数据和消息结构错误 |
| Vite | 构建工具 | 开发体验好，HMR 快 |
| Tailwind CSS | 样式 | 快速实现暗色主题和标签布局 |
| localStorage | 本地存储 | MVP 不需要账号和数据库 |
| FastAPI | 后端 | 适合 AI 接口代理，开发快 |
| Pydantic | 校验 | 校验 AI 输出 JSON |
| Vercel | 前端部署 | 免费额度够用，自动 CI/CD |

## 6. 标签系统设计

### 6.1 双层结构：方向条 + 词云

灵感生成的入口分为两层，方向条决定"频道"，词云决定"内容"。

```text
方向条（Direction Bar）
  ├─ 角色设计    Character
  ├─ 场景设计    Scene
  ├─ 世界观设计  Worldview
  └─ 故事梗概    Plot
        ↓ 决定词云的 90%
词云（Tag Cloud）
  ├─ 90% 当前方向词
  └─ 10% 跨界惊喜词（带角标，点击跳转对应方向）
```

### 6.2 方向定义

每个方向对应独立词库 + 独立 AI prompt 链路。

| 方向 | 关键字段 | 词性偏向 | 示例 |
|---|---|---|---|
| 👤 角色设计 | `character` | 身份 / 性格 / 关系 / 秘密 | 孤儿、双重人格、童年阴影、私生子、社恐 |
| 🎬 场景设计 | `scene` | 地点 / 时间 / 天气 / 氛围 | 废墟、雨夜、深海实验室、霓虹巷、黎明 |
| 🌍 世界观设计 | `worldview` | 时代 / 规则 / 体系 / 危机 | 末世、蒸汽朋克、灵能觉醒、灵媒制、轮回律 |
| 📖 故事梗概 | `plot` | 事件 / 冲突 / 结构 / 转折 | 背叛、重逢、献祭、时间循环、复仇 |

### 6.3 标签分类（细分维度）

每个方向下的词进一步标记 `category`，支持词云内的混合多样性。

```text
- 地点 location
- 角色性格 character
- 情节 plot
- 时间跨度 timespan
- 故事类型 genre
- 视觉元素 visual
- 情绪氛围 mood
```

### 6.4 词云生成规则

每次刷新词云时，固定输出 15-25 个标签，构成如下：

| 来源 | 比例 | 说明 |
|---|---|---|
| 当前方向词 | 90% | 来自该方向词库 + AI 基于已选词的方向内联想 |
| 跨界惊喜词 | 10% | 来自其他方向词库；带角标，点击触发方向跳转 |

其中当前方向词内部进一步分配：

- **本地词库随机抽取**：约 60%，零延迟，保证基本盘
- **AI 动态生成**：约 40%，基于已选词做关联推荐

首批进入对话框时（默认方向为「角色设计」），全部从本地词库抽取，无需等待 AI；用户切换方向或点击「换一批」之后，混合 AI 生成。

### 6.5 方向切换与已选词联动

切换方向时词云立即重排，已选词常驻锁定栏，并反向影响新方向的词云内容。

```text
锁定栏：[双重人格] [童年阴影]   ← 角色方向选的词
切到「场景设计」
  → 后端收到 direction=scene + selectedTags=[双重人格, 童年阴影]
  → 返回偏向"适合这种角色"的场景词：
     旧公寓、老相册、镜子房间、童年的家、梅雨季
```

跨界词的处理：

- 词云中 10% 的跨界词带方向小角标（🎬 / 🌍 / 📖 / 👤）
- 用户点击跨界词时：先把该词加入锁定栏，然后自动切换方向条到对应方向，词云重排

### 6.6 标签数据结构

```ts
type Direction = 'character' | 'scene' | 'worldview' | 'plot';

type TagCategory =
  | 'location'
  | 'character'
  | 'plot'
  | 'timespan'
  | 'genre'
  | 'visual'
  | 'mood';

interface Tag {
  id: string;
  text: string;
  direction: Direction;        // 该词所属方向
  category: TagCategory;       // 细分维度
  source: 'local' | 'ai';      // 来源
  isCrossover?: boolean;       // 是否跨界词（用于角标显示）
}
```

## 7. 后端设计

### 7.1 后端职责

```text
1. 保管 AI API Key
2. 按方向（direction）路由到对应词库与 prompt 链路
3. 生成词云：90% 当前方向词 + 10% 跨界词，结合已选词反向影响
4. AI 推荐标签（基于已选标签 + 当前方向做关联推荐）
5. 基于用户选词生成故事梗概（200-300 字）
6. 处理用户在对话框中的修改需求
7. Prompt 管理（每个方向独立 prompt 模板）
8. JSON 校验
9. 限流与鉴权
```

## 9. 数据结构

### 9.1 TypeScript 类型

```ts
// 方向
type Direction = 'character' | 'scene' | 'worldview' | 'plot';

// 标签细分维度
type TagCategory = 'location' | 'character' | 'plot' | 'timespan' | 'genre' | 'visual' | 'mood';

// 标签
interface Tag {
  id: string;
  text: string;
  direction: Direction;          // 该词所属方向
  category: TagCategory;         // 细分维度
  source: 'local' | 'ai';
  isCrossover?: boolean;         // 是否跨界词（带角标显示）
}

// 消息
interface ChatMessage {
  id: string;
  role: 'muse' | 'user';
  content: string;
  wordCount?: number;
  createdAt: string;
}

// 会话
interface ChatSession {
  id: string;
  currentDirection: Direction;       // 方向条当前所在方向
  visitedDirections: Direction[];    // 用户走过的方向顺序
  selectedTags: Tag[];               // 锁定栏内的已选词（跨方向常驻）
  excludeTexts: string[];            // 本会话已展示过的词，去重用
  messages: ChatMessage[];
  createdAt: string;
  updatedAt: string;
}
```

## 10. AI Prompt 设计

### 10.1 词云生成（按方向路由）

每个方向走独立 prompt 模板，输入参数：当前方向、已选词、需要生成的数量。

通用要求：

```text
1. 90% 的词必须属于当前方向，10% 必须属于其他方向并标记 isCrossover=true
2. 已选词反向影响：词的内容要"适合 selectedTags"，体现已选词的痕迹
3. 不要重复 selectedTags 和 excludeTexts 中的任何词
4. 每个词 2-6 个字，简洁有画面感
5. 每个词附带 direction 和 category 字段
6. 严格返回 JSON
```

各方向词性偏向：

| 方向 | Prompt 词性引导 |
|---|---|
| character | 突出"人"——身份、性格、关系、秘密、动机；避免单纯地点 / 时间词 |
| scene | 突出"地+时+氛围"——地点、时间、天气、视觉、声音 |
| worldview | 突出"世界规则"——时代、体系、危机、势力、异常法则 |
| plot | 突出"事件"——冲突、转折、动机、目标、结局基调 |

跨界词的挑选规则：

```text
- 在已选词最匹配的"另一个方向"里挑（例：选了角色就推一个适配的场景）
- 跨界词同样要受 selectedTags 影响，体现连贯性
- 跨界词的 isCrossover 必须为 true，前端用于显示角标和触发跳转
```

### 10.2 生成故事梗概

```text
要求：
1. 基于用户选择的词汇组合，生成一段故事情节梗概
2. selectedTags 中每条带有 direction，按方向归类阅读：
   - character 方向的词作为主角设定
   - scene 方向的词作为场景与氛围
   - worldview 方向的词作为背景规则
   - plot 方向的词作为冲突与事件
3. 故事中每一类词都要在文本里"看得见痕迹"，让用户感觉"这是我抽出来的故事"
4. 字数控制在 200-300 字
5. 要有画面感，像在描述一个电影片段
6. 有明确的主角、场景、冲突
7. 结尾留有悬念或余韵，不要写完整结局
8. 语言风格：简洁、有节奏感、小学生也能读懂
9. 不要解释创作思路，直接输出故事
```

### 10.3 修改 / 续写故事

```text
要求：
1. 基于原始故事和用户的修改需求，输出修改后的完整故事
2. 保持 200-300 字
3. 尊重用户的修改意图
4. 保持故事的连贯性和画面感
5. 直接输出修改后的故事，不要解释改了什么
```
